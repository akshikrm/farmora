## Pull Request Policy

- All changes must be submitted via Pull Request (PR). Direct pushes to protected branches are not allowed.
- PR checklist:

  - Keep PRs focused and small where possible.
  - Describe the change and link any related issue.
  - Add or update tests and documentation as appropriate.
  - Ensure CI passes before requesting review.
  - Maintainers will review and merge; follow review feedback.
